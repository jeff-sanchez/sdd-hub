O que é
Spec-Driven Development
Uma metodologia que substitui o caos do vibe coding por contratos escritos antes do código. Cada fase produz um entregável. A fase seguinte só começa quando o entregável anterior está aprovado.
O problema que o SDD resolve
O vibe coding cria um loop sem fim: você descreve o que quer, a IA gera código, não funciona como esperado, você reescreve o prompt, recebe código diferente, e o ciclo continua. Cada iteração pode introduzir regressões no que já funcionava. O resultado é um sistema que ninguém entende completamente — nem você, nem a IA.
A diferença fundamental
No SDD, o código é a última etapa, não a primeira. Você define o problema, levanta requisitos, fecha o escopo, projeta a arquitetura — e só então pede código. Isso elimina alucinações porque a IA não está inventando: está implementando uma especificação.

Fase 0 começa aqui. Preencha o briefing com precisão — um problem statement vago é a causa mais comum de projetos que travam no PRD.
Evite generalidades como "melhorar a gestão". Prefira: "Os instrutores precisam ligar para a secretaria para saber sua carga horária, o que leva 20min por consulta."
Deve ter um número ou critério verificável. Sem isso, não existe definição de "pronto".
Restrições descobertas durante a implementação são 10x mais caras que as identificadas aqui.
